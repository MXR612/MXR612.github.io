A 这是一棵树吗?

题意

给出一张图nnn个点的度数, 问是否可能是一棵树.

故事

超级大水题, 显然一棵树上有n−1n-1n−1条边, 每加入一条边会使整张图的度数和增加222. 判断度数和是否为2n−22n-22n−2, 特判如果n>2n>2n>2, 任一点的读书不能为0.

#include <stdio.h>

long long n, s;
bool pos = true;

signed main() {
#ifndef ONLINE_JUDGE
    freopen("A.in", "r", stdin);
#endif

    scanf("%lld", &n);
    for (long long i = 0, a; i < n; ++i)
        scanf("%lld", &a), pos &= (a > 0), s += a;

    if (s == (n - 1) * 2 && (n == 1 || pos))
        printf("Possible");
    else
        printf("Impossible");

    return 0;
}

B 选数排列

题意

给出nnn个数, 从中选出RRR组, 每组CCC个. 每组的贡献是其中的最大值减最小值, 问这RRR组的最大值最小是多少.

故事

还是那句话: 不贪心的人在比赛中也不会贪心.

为什么我一直对贪心的正确性那么怀疑呢? 还是因为对没能证明的东西总会有莫名的恐惧?

最后我打了O(n2)O(n^2)O(n2)的暴力, 因为数据年代久远所以拿了80? (这次我没在暴力外面套数据范围判断)

一个显然的结论: 每次选的数比是排完序后数组内的一段数. 如果有两组数所属的区间相交, 肯定可以通过交换元素使两者的最大值/最小值变优.

设fi,jf_{i,j}fi,j​为选了iii组, 选到了第jjj个数, 每组贡献的最大值的最小值. 转移如下:

fi,j=mink=0k≤j−Cfi−1,k + Pj−Pj−Cf_{i,j}=min_{k=0}^{k \leq j-C}f_{i-1,k} \ + \ P_j - P_{j-C}fi,j​=mink=0k≤j−C​fi−1,k​ + Pj​−Pj−C​

#define MXN (500020)

#include <stdio.h>
#include <string.h>

#include <algorithm>

int N, R, C;
int P[MXN];

// Sub1
int a[MXN], f[1024][1024], min, ans = 1e9;

signed main() {
#ifndef ONLINE_JUDGE
    freopen("B.in", "r", stdin);
#endif

    scanf("%d%d%d", &N, &R, &C);
    for (int i = 0; i < N; ++i)
        scanf("%d", &P[i]);

    std::sort(P, P + N);

    for (int i = C - 1; i < N; ++i)
        a[i] = P[i] - P[i - C + 1];

    memset(f, 0x3f, sizeof(f));
    for (int i = 0; i <= N; ++i)
        f[0][i] = 0;
    for (int i = 1, j; i <= R; ++i)
        for (min = 1e9, j = C - 1; j < N; ++j)
            f[i][j] = std::max(min = std::min(min, f[i - 1][j - C]), a[j]);
    for (int i = 0; i < N; ++i)
        ans = std::min(ans, f[R][i]);

    printf("%d", ans);

    return 0;
}

题解

二分答案+贪心判断
跟前几天那个B......组题几乎一样, 我也几乎一样的不会.

我们二分出每组的最大和, 然后贪心地选出尽量多组数.
